# Politica di Sicurezza e Stato del Progetto 🛡️

## Stato dello Sviluppo
QuizMaker è attualmente in **fase di sviluppo attivo (Beta)**. Sebbene sia funzionale e pronto all'uso, potrebbero esserci cambiamenti strutturali frequenti. 

L'applicazione è interamente **client-side**: i dati dei quiz (JSON) vengono elaborati direttamente nel tuo browser e non vengono inviati ad alcun server esterno (a meno che tu non scelga di caricare dataset pubblici dal catalogo).

## Versioni Supportate
Attualmente, forniamo aggiornamenti di sicurezza e correzioni solo per l'ultima versione disponibile sul branch `main`.

| Versione | Supportata |
| :--- | :--- |
| Branch `main` | ✅ Si |
| Versioni precedenti | ❌ No |

## Segnalazione di Vulnerabilità
Vogliamo mantenere QuizMaker sicuro per tutti. Se trovi una vulnerabilità di sicurezza:

1.  **NON aprire una Issue pubblica.**
2.  Invia una segnalazione privata tramite il sistema di [GitHub Private Vulnerability Reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability).
3.  In alternativa, contatta il manutentore principale del progetto.

Cercheremo di rispondere entro 48-72 ore e di risolvere il problema il prima possibile.

## Aggiornamenti delle Dipendenze
Utilizziamo strumenti automatizzati (come Dependabot) per monitorare e aggiornare le dipendenze di Jekyll e Ruby Gem per garantire che l'ambiente di build sia sempre sicuro.
