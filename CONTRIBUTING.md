# Contribuire a QuizMaker 🧠

Grazie per l'interesse nel contribuire a QuizMaker! Questo progetto è nato per rendere l'apprendimento più accessibile e divertente, e il tuo aiuto è prezioso.

## 🌟 Come puoi contribuire?

Esistono diversi modi per aiutare il progetto, dai più semplici ai più tecnici.

### 1. Segnalare Bug o Suggerire Funzionalità
Se trovi qualcosa che non funziona o hai un'idea brillante:
- Apri una [Issue](https://github.com/Ste517/QuizMaker/issues/new/choose).
- Utilizza i **Form strutturati** che abbiamo preparato: ti guideranno nel fornirci tutte le informazioni necessarie.

### 2. Aggiungere Nuovi Dataset (Il modo più utile!)
Il cuore di QuizMaker sono i suoi quiz. Puoi aggiungere i tuoi:
1.  Usa l'**Editor Integrato** (`editor.html`) per creare il tuo file JSON.
2.  Assicurati che ogni domanda abbia una spiegazione (aiuta molto chi studia!).
3.  Copia il file JSON nella cartella `data/user_made/`.
4.  Aggiorna il file `data/index.json` aggiungendo la tua nuova creazione alla lista.
5.  Invia una Pull Request!

### 3. Migliorare il Codice
Se sei uno sviluppatore:
- **Vanilla JS**: Cerchiamo di mantenere il progetto leggero, quindi usa JavaScript puro senza framework pesanti.
- **Tailwind CSS**: Il design è basato su Tailwind. Cerca di mantenere la coerenza visiva ("Liquid Glass").
- **Jekyll**: Il sito è generato con Jekyll. Se modifichi la struttura, assicurati che il comando `bundle exec jekyll serve` non dia errori.

## 🚀 Processo di Pull Request

1.  Fai un **Fork** del repository.
2.  Crea un branch per la tua modifica (`git checkout -b feature/nome-fantastico`).
3.  Fai il **Commit** dei tuoi cambiamenti (`git commit -m 'Aggiunta funzionalità X'`).
4.  Fai il **Push** sul branch (`git push origin feature/nome-fantastico`).
5.  Apri una **Pull Request** spiegando chiaramente cosa hai fatto e perché.

## 📜 Codice di Condotta
Partecipando a questo progetto, accetti di seguire il nostro [Codice di Condotta](CODE_OF_CONDUCT.md). Sii gentile e rispettoso verso tutti i membri della community.
